Exercise 1: LoggingExample
  console output (default Logback pattern, roughly):
    HH:mm:ss.SSS [main] ERROR LoggingExample - This is an error message
    HH:mm:ss.SSS [main] WARN  LoggingExample - This is a warning message

Exercise 2: ParameterizedLoggingExample
  console output:
    HH:mm:ss.SSS [main] INFO  ParameterizedLoggingExample - User Asvathama logged in
    HH:mm:ss.SSS [main] WARN  ParameterizedLoggingExample - User Asvathama failed login 3 times
  -- the {} placeholders get filled in with username and loginAttempts in order

Exercise 3: AppenderLoggingExample (using logback.xml, root level = debug)
  printed to BOTH console and app.log file (same content, same pattern):
    HH:mm:ss.SSS [main] DEBUG AppenderLoggingExample - debug message, only goes to file and console since root level is debug
    HH:mm:ss.SSS [main] INFO  AppenderLoggingExample - application started
    HH:mm:ss.SSS [main] ERROR AppenderLoggingExample - something went wrong while processing

  -- app.log is created in the project's working directory after this runs
